Lazy loading

This section describes how battle-backbone allows you to use lazy loading for your application

Introduction

Lazy loading makes an application faster and more modular because the client only loads resources when they are really required. battle-backbone provides you with two lazy loading mechanism out of the box.

Lazy module loading

The navigation services provides you with lazy loading of modules where you bind a module controller to a route. The module and all of its dependencies will not be loaded until the route is requested.

More information: Lazy module loading

Lazy resource loading

In addition to loading modules dynamically you can also load resources when a function is invoked. This allows you to write a multi-purpose controller that loads its resources only when they are required.

More information: Lazy resources loading

Combined

You usually combine these two approaches when writing modules:

  • Module bundles are loaded lazily using the navigation service and the configuration file.
  • When the module is loaded an event will be triggered on it. The function invoked by this event will load the required resources.

This allows you to write a controller that handles multiple routes (e.g. login and logout) but loads the resources for every route only if they are really required.
