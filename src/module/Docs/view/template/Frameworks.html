Third-party components

This section describes what third party components are included in battle-backbone

Introduction

Since battle-backbone is neither a library or a framework but a toolbox that provides you with battle-tested best practices and common development patterns for single page applications it contains third-party dependencies. This page gives you a list of libraries and frameworks used by battle-backbone and describes reasons why they are included.

Backbone

Backbone is the core framework that powers battle-backbone. We use it for our applications since it is small, understandable, very well documented and can be extended easily.

Marionette

Backbone provides only a few things and you often need complex structures like nested views or views that render collections. Marionette provides all this and adds useful components like a modularization interface and behaviors.

Backbone.Radio

Marionette comes with Backbone.Wreqr as event system. Backbone.Radio provides basically the same but uses a cleaner and more intuitive interface and replaces Wreqr in battle-backbone.

lodash

Backbone usually uses underscore as utility and abstraction library. It has been replaced with the API-compatible lodash that provides more functionality and is believed to be faster in some cases.

jQuery

In addition to underscore (or lodash) Backbone needs jQuery or an API compatible library like zepto. We use jQuery here because it can be integrated easily and provides many things developers use regularily. It also has a large plugin base

require.js

JavaScript does not provide a module system that can be used by browsers today and ES5 doesn't provide a module system at all. While there are a couple of module system for front-end development require.js provides us with bundling, plugins and dynamic loading of resources.

require-css

This plugin for require.js enables modules to load their own CSS without being forced to write on large centralized CSS file.

require.js i18n

This plugin makes it easy to load localization files based on the currently set language without the need to write complex loading logic somewhere.

Grunt

Grunt is used as task runner because it has lots of plugins, is easy to understand for new team members and it supports the require.js optimizer r.js without any hacks.
You can find more information in the Toolchain section.

CoffeeScript

Originally, battle-backbone was written in CoffeeScript because it provides a neat syntax that is easy to read and understandle and looks a lot less messy than JavaScript in many cases.
However, since CoffeeScript cannot be used everywhere there is also a native JavaScript version of battle-backbone that is not compiled from CoffeeScript but manually written.
